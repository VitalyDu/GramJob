## 🔴 CRITICAL

### BUG-01: `seed-permissions.ts` — неверные имена action-ов для favorites, saved-searches, blocks

**Файл:** `backend/src/scripts/seed-permissions.ts`

**Симптом:** `GET /api/favorites`, `GET /api/saved-searches`, `GET /api/blocks` возвращают `403 Forbidden` для авторизованного пользователя. Страницы `/dashboard/favorites`, `/dashboard/saved-searches`, `/dashboard/blocks` не загружают данные.

**Причина:** Имена action-ов в `AUTHENTICATED_PERMISSIONS` не совпадают с именами handler-ов в контроллерах:

| В seed (неверно)                        | Фактический handler | Правильный action                         |
| --------------------------------------- | ------------------- | ----------------------------------------- |
| `api::favorite.favorite.find`           | `findMine`          | `api::favorite.favorite.findMine`         |
| `api::favorite.favorite.delete`         | `remove`            | `api::favorite.favorite.remove`           |
| `api::saved-search.saved-search.find`   | `findMine`          | `api::saved-search.saved-search.findMine` |
| `api::saved-search.saved-search.delete` | `remove`            | `api::saved-search.saved-search.remove`   |
| `api::block.block.find`                 | `findMine`          | `api::block.block.findMine`               |
| `api::block.block.delete`               | `remove`            | `api::block.block.remove`                 |

**Исправление:**

В `backend/src/scripts/seed-permissions.ts` заменить:

```ts
// было
'api::favorite.favorite.find',
'api::favorite.favorite.create',
'api::favorite.favorite.delete',
'api::saved-search.saved-search.find',
'api::saved-search.saved-search.create',
'api::saved-search.saved-search.delete',
'api::block.block.find',
'api::block.block.create',
'api::block.block.delete',

// стало
'api::favorite.favorite.findMine',
'api::favorite.favorite.create',
'api::favorite.favorite.remove',
'api::saved-search.saved-search.findMine',
'api::saved-search.saved-search.create',
'api::saved-search.saved-search.remove',
'api::block.block.findMine',
'api::block.block.create',
'api::block.block.remove',
```

После исправления нужно перезапустить backend — `seedPermissions` применится автоматически.

**Примечание:** Этот баг существовал до редизайна (Sprint 5). Redizain его не вводил и не устранял.

## 🔴 CRITICAL (Frontend)

### BUG-04: `<Script strategy="beforeInteractive">` вне `<body>` — консольные ошибки и hydration mismatch

**Файл:** `frontend/src/app/layout.tsx`  
**Статус: ✅ ИСПРАВЛЕНО**

**Симптомы:**

- `Cannot render a sync or defer <script> outside the main document without knowing its order`
- `<html> cannot contain a nested <script>. This will cause a hydration error.`
- Hydration mismatch: Telegram SDK устанавливает `style="--tg-viewport-height:100vh; --tg-viewport-stable-height:100vh"` на `<html>` до гидратации, сервер рендерит `<html>` без этих стилей.

**Причина:**

1. `<Script>` стоял как прямой дочерний элемент `<html>` (между `<html>` и `<body>`), что является невалидным HTML — Next.js 15 / Turbopack отвергает такой порядок.
2. Telegram SDK модифицирует `document.documentElement.style` при загрузке, что не совпадает с SSR-рендером `<html>`.

**Исправление:**

```tsx
// было
<html lang="ru" className={inter.variable}>
  <Script src="https://telegram.org/js/telegram-web-app.js" strategy="beforeInteractive" />
  <body ...>

// стало
<html lang="ru" className={inter.variable} suppressHydrationWarning>
  <body ...>
    <Script src="https://telegram.org/js/telegram-web-app.js" strategy="beforeInteractive" />
```

`suppressHydrationWarning` на `<html>` подавляет ложное предупреждение об атрибуте `style`, который Telegram SDK добавляет на стороне клиента.

## 🟡 MINOR

### BUG-03: Отсутствует `favicon.ico`

**Симптом:** `GET /favicon.ico` → 404. В браузере нет иконки вкладки.

**Причина:** В `layout.tsx` указан `icons: { icon: '/logo-vertical.png' }`, но браузеры также запрашивают `favicon.ico` напрямую. Файл `favicon.ico` не размещён в `frontend/public/`.

**Исправление:** Добавить `favicon.ico` в `frontend/public/` (можно сконвертировать из `logo-vertical.png`), или добавить в `layout.tsx`:

```ts
icons: {
  icon: '/logo-vertical.png',
  shortcut: '/logo-vertical.png',
}
```
