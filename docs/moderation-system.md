# GramJob — Moderation System

---

## Сущности, проходящие модерацию

1. **Вакансии** — все, включая внешние (после парсинга)
2. **Резюме**
3. **Компании**

---

## Статусы и переходы

### Вакансии

```
draft → moderation → published
                  → rejected
published → archived  (вручную пользователем)
published → expired   (автоматически через 60 дней)
rejected  → draft     (пользователь редактирует и пересылает)
expired   → draft     (можно переопубликовать, тратятся кредиты)
```

### Резюме

```
draft → moderation → published
                  → rejected
published → archived  (вручную)
rejected  → draft     (пересылка после правок)
```

### Компании

```
draft → moderation → published
                  → rejected
rejected → draft    (пересылка после правок)
```

---

## Воркфлоу модерации

### 1. Подача на модерацию

Пользователь нажимает «Опубликовать» → статус меняется на `moderation` → пользователь видит «На модерации».

Для публикации вакансии тратится 1 кредит / лимит плана в момент перехода в `moderation`.

### 2. Рассмотрение модератором

Модератор работает в Strapi Admin Panel:
- Видит очередь на модерацию (отфильтровано по `status = moderation`)
- Просматривает содержимое
- Принимает решение: **одобрить** или **отклонить**

### 3. Одобрение

- Статус → `published`
- Для вакансий: устанавливается `expiresAt = now + 60 days`
- Пользователь получает Telegram-уведомление

### 4. Отклонение

- Статус → `rejected`
- Модератор обязательно указывает причину отклонения
- Пользователь получает Telegram-уведомление с причиной
- Для вакансий: кредит/лимит **возвращается** пользователю (TBD — уточнить правило)

---

## Причины отклонения

| Код | Описание |
|-----|----------|
| `spam` | Спам или дублирующийся контент |
| `fake` | Фиктивная вакансия/компания |
| `inappropriate` | Неприемлемый контент |
| `incomplete` | Недостаточно информации |
| `wrong_category` | Неправильная категория |
| `salary_mismatch` | Некорректные данные о зарплате |
| `contact_info` | Контактные данные в запрещённых полях |
| `other` | Другое (с обязательным комментарием) |

---

## Правила для вакансий

Требования к публикации:
- Заполнены обязательные поля: title, description, employmentType, workFormat, country
- Зарплата: если salaryFrom указана, должна быть > 0
- Описание: минимум 100 символов
- Не содержит запрещённого контента (дискриминация, нелегальная деятельность)
- Контактные данные не размещены в полях description/requirements (только через официальный механизм)

Запрещено:
- Вакансии МЛМ, финансовые пирамиды
- Вакансии без чёткого описания обязанностей
- Вакансии с явно нереальными условиями

---

## Правила для резюме

Требования:
- Заполнены: firstName, lastName, title, country, about (минимум 50 символов)
- Указан хотя бы один навык
- Не содержит контактных данных вне поля contacts

---

## Правила для компаний

Требования:
- Заполнены: name, description (минимум 100 символов), country
- Загружен логотип
- Указан сайт или Telegram

---

## Повторная подача

После отклонения пользователь может:
1. Исправить контент (статус → `draft` автоматически при редактировании отклонённой записи)
2. Повторно подать на модерацию
3. Ограничений на количество повторных подач нет

---

## SLA модерации

- Целевое время: **24 часа** в рабочие дни
- Для срочных вакансий (urgent): приоритет в очереди
- Уведомление пользователю если модерация затянулась > 48 часов (TBD)

## Автоматическая модерация (будущее)

На текущем этапе модерация ручная. В будущем возможно:
- Автоматическая проверка на спам (ML)
- Проверка ключевых слов
- Авто-одобрение для верифицированных компаний

---

## Жалобы и связь с модерацией

Жалоба пользователя (Report) попадает в очередь на рассмотрение модераторами.
При подтверждении жалобы — сущность переводится в `rejected` или удаляется.

---

## Инструменты модератора (Strapi Admin)

- Фильтр по status = moderation
- Просмотр сущности с Preview
- Кнопки: Approve / Reject (с выбором причины)
- История модерации (кто, когда, решение)
- Статистика: сколько на проверке, среднее время
